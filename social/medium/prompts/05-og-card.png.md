## Medium Post 5 — OG Card

**Aspect ratio:** 1200×630

**Concept:** Split screen. Left half: a code editor showing React hooks (useQuery, useMutation). Right half: a rendered todo list UI with checkboxes. The two halves are connected by a subtle glow line showing the data flow from code → UI.

**Logo:** Top-left corner. ctrodb wordmark (user uploads the logo file separately).

**Composition:**
- Left 50%: code editor mockup. Dark background (#1e293b). Several lines of monospace text suggesting `useQuery("todos", ...)` and `useMutation("todos")`. The code doesn't need to be perfectly readable — just recognizable as React hooks in a dark editor. The `useQuery` line is highlighted in cyan. Line numbers on the left gutter in dim gray.
- Center: a thin vertical glowing cyan line (#06b6d4) with a subtle blur/glow effect, connecting the two halves
- Right 50%: a simple todo list UI mockup. 4-5 horizontal items, each with a small square checkbox and text. First item has a checked square with strikethrough text. Remaining items are unchecked. The UI is clean — white background, dark text, minimal borders. The overall shape suggests a phone or narrow panel.
- Bottom area spanning both halves: headline "Build a Reactive Todo App Without Redux or Zustand" in white sans-serif, 32pt, left-aligned with 60px left margin

**Colors:** Editor background dark (#1e293b), UI white (#f8fafc), accent cyan (#06b6d4), code text green (#22c55e for strings), line numbers (#475569).

**Typography:** Headline Inter or similar geometric sans-serif, bold, 32pt. Code in JetBrains Mono. UI text in system sans-serif.

**Technical notes:** 1200×630. Logo top-left (user uploads separately). The todo list UI should look like a real component — don't over-design it, keep it simple. The glow line in the center should be subtle, not neon.
