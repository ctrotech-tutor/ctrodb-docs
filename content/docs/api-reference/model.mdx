## Constructor

Internal — models are created by `Collection._toModel(){:ts}`.

`Model{:ts}` uses a `Proxy{:js}` to expose record fields as direct properties.

## Properties

```ts
model.id: ID
```

All record fields are accessible as properties:

```ts
user.name
user.email
// etc.
```

## Methods

### update()

```ts
update(changes: Partial<T>): Promise<Model<T>>
```

Calls `collection.update(this.id, changes){:ts}`.

### delete()

```ts
delete(): Promise<void>
```

Calls `collection.delete(this.id){:ts}`.

### toJSON()

```ts
toJSON(): T
```

Returns a plain copy of the data — no methods or proxy.

## Relation getters

Auto-attached when the schema defines relations and the relations plugin is active.
Each getter returns a `QueryBuilder{:ts}` pre-filtered to match the relation.
