title: Query Engine
description: Filter, sort, and paginate records
---

`QueryBuilder{:ts}` provides a fluent API for reading data. It does not support joins —
use [relations](/docs/plugins/relations) for related data.

## Basic usage

```ts
const results = await collection
  .query()
  .where("status", "==", "active")
  .sort({ createdAt: "desc" })
  .limit(10)
  .fetch()
```

## where

```ts
query.where(field, operator, value)
query.where(field, value) // shorthand for ==
```

Supported operators: `==`, `!=`, `>`, `<`, `>=`, `<=`

```ts
query.where("age", ">=", 18)
query.where("status", "active")       // shorthand
query.where("name", "!=", "admin")
```

## orWhere

```ts
query
  .where("role", "==", "admin")
  .orWhere((q) => q.where("role", "==", "moderator"))
```

Groups conditions with OR logic. Each `orWhere` block is one group.

## sort

```ts
query.sort({ fieldName: "asc" | "desc" })
```

Only one sort field per query. Passing multiple entries picks the first.

```ts
query.sort({ createdAt: "desc" })
```

## limit and offset

```ts
query.limit(10)
query.offset(20).limit(10)
```

Use `offset` with `limit` for pagination. Offset skips records before slicing.

## search

```ts
query.search("title", "database")
```

Case-insensitive substring search (`String.includes(){:js}`). The [FTS plugin](/docs/plugins/full-text-search)
adds an inverted index, accessible via `FTSIndexer.search(){:ts}`.

## Terminal methods

```ts
await query.fetch()              // Model<T> & T[]
await query.first()              // Model<T> & T | undefined
await query.count()              // number
await query.toArray()            // T[] (plain objects)
```

`first(){:ts}` is equivalent to `.limit(1).fetch()` then returning `results[0]`.

## How queries execute

The `QueryPlanner{:ts}` examines conditions and indexes to pick a strategy:

| Strategy | When | How |
|----------|------|-----|
| `index_scan{:ts}` | Field has an index | Uses IndexedDB index with key range |
| `id_lookup{:ts}` | Querying by `id{:ts}` field with `==` | Direct adapter lookup |
| `full_scan{:ts}` | No index match | Iterates all records, filters in memory |

Sort, offset, and limit are applied in memory after fetching.

## Sorting behavior

Uses JavaScript's `<` and `>` comparison. Mixed types compare via `<`/`>` rules —
numbers sort numerically, strings lexicographically.
