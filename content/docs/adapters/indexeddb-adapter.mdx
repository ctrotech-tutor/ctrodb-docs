---
title: IndexedDBAdapter
description: Persistent storage for browsers
---

`IndexedDBAdapter{:ts}` uses the browser's IndexedDB API. Data persists across sessions.

## How it works

- Each collection is an object store with `id{:ts}` as keyPath
- Auto-increment is handled by IndexedDB
- Indexes are created during migration (`onupgradeneeded{:js}`)
- A `_ctrodb_meta{:ts}` store holds schema version and metadata

## Schema migration

When you change the schema version, IndexedDB fires `onupgradeneeded{:js}`.
The adapter creates new object stores and indexes:

```ts
const db = new Database({
  schema: {
    version: 2, // bump to trigger migration
    collections: {
      todos: {
        fields: { title: { type: "string" } },
        indexes: [{ field: "title" }],
      },
    },
  },
})
```

New collections and indexes are added. Existing data is preserved.

## Browser support

All modern browsers (Chrome, Firefox, Safari, Edge) support IndexedDB.
The adapter checks for `window.indexedDB{:js}` at runtime.

## Transaction behavior

Uses `IDBTransaction{:ts}` with `readwrite{:txt}` mode across all object stores.
If the callback throws, `tx.abort(){:ts}` is called.

## Limitations

- Not available in Node.js
- IndexedDB size limits vary by browser (typically 50MB–unlimited)
- Blocked on database version change (tab conflict)
